\section{Primeros pasos con Apache Kafka}

En esta sección se explica cómo se ha resuelto el primer ejercicio. El primer paso consiste en acceder al contenedor donde reside el servicio de Kafka para ejecutar las herramientas de línea de comandos integradas.

\subsection{Acceso al entorno y Gestión de Tópicos}

Para comenzar la interacción, nos conectamos al contenedor mediante el comando \texttt{docker exec}, lo que nos permite abrir una terminal interactiva dentro del nodo de Kafka:

\begin{vscodeTerminal}
docker exec -it kafka-backend bash
\end{vscodeTerminal}

Una vez dentro, el objetivo es crear nuestro primer canal de comunicación o "Topic". Los tópicos son las categorías o nombres de feeds donde se almacenan y publican los registros de datos.

\subsubsection{Creación y Verificación}

Utilizamos el script \texttt{kafka-topics.sh} para definir una nueva estructura llamada \texttt{firstTopic}. Los comandos ejecutados y su función se resumen en la siguiente tabla:

\texttt{--bootstrap-server} & Dirección del servidor Kafka inicial para establecer conexión. \\ \hline
\texttt{--create} & Instrucción para generar un nuevo tópico. \\ \hline

\subsection{Efecto de la propiedad \textit{auto\_offset\_reset}}

Esta propiedad determina qué debe hacer el consumidor cuando se conecta a un tópico de Kafka y es la primera vez que se conecta o el offset actual inválido (los datos se han borrado por la política de retención). 

Si aplicamos la configuración \textit{earliest} es consumidor retrocede al inicio del log y lee todos los mensajes históricos que estén disponibles por el broker. Es la configuración más útil cuando se desea evitar perder cualquier dato tras una caída del sistema.

La configuración \textit{latest} ignora los mensajes antiguos y se queda escuchando solamente los que se produzcan desde su conexión. Es una configuración pensada para la monitorización puramente en tiempo real donde los datos del pasado ya no son relevantes para la visualización actual.


\subsection{Intentando enviar datos no serializables}

Podemos intentar enviar estructuras de datos complejas como \textit{arrays} de \textit{NumPy} de la siguiente manera:

\begin{vscodePython}
# Create Kafka producer
producer = KafkaProducer(
    bootstrap_servers=kafka_bootstrap_servers,
    value_serializer=lambda v: json.dumps(v).encode('utf-8')
)

print(f"Producing messages to Kafka topic '{kafka_topic}'...")

try:
    while True:
        # Generate sensor data
        sensor_data = generate_sensor_data()

        # Publish sensor data to Kafka
        data = np.array([e for e in sensor_data.values()])
        print(f"{data=}")
        producer.send(kafka_topic, data)

\end{vscodePython}

Al hacer esto obtendremos un error como el siguiente:
\begin{vscodeTerminal}
Producing messages to Kafka topic 'water_quality'...
data=array([1.77392590e+09, 2.63006815e+01, 8.27793620e+00, 7.57000000e+00, 1.05800000e+01])
----------------------------------------------------
TypeError                                 Traceback (most recent call last)
Cell In[15], line 22
     20 data = np.array([e for e in sensor_data.values()])
     21 print(f"{data=}")
---> 22 producer.send(kafka_topic, data)
     24 print(f"Sent: {sensor_data}")
     26 # Wait for 1 second

TypeError: Object of type ndarray is not JSON serializable
\end{vscodeTerminal}

Este error indica que el objeto de tipo \texttt{ndarray} no puede ser serializado mediante JSON. El motivo es que el serializador configurado (\texttt{json.dumps}) solo admite tipos nativos de Python (como \texttt{int}, \texttt{float}, \texttt{str}, \texttt{list}, \texttt{dict}, etc.), pero no estructuras propias de bibliotecas externas como \texttt{NumPy}.

Para resolver este problema existen varias estrategias:

\begin{itemize}
    \item Convertir explícitamente el array a un tipo serializable (por ejemplo, una lista de Python).
    \item Definir un serializador personalizado que transforme objetos de \texttt{NumPy}.
    \item Utilizar formatos de serialización alternativos (como Avro o Protobuf) que soporten esquemas más ricos.
\end{itemize}

\subsection{Crear varios productores}

Para crear múltiples productores, se han realizado varios cambios. En primer lugar, hemos interpretado \textit{generar varios productores} como instanciar nuevos objetos de la clase \textit{KafkaProducer}; no solo usar una sola instancia y enviar mensajes con \textit{keys} diferentes. Por tanto el proceso consiste en:
\begin{lstlisting}
1. Elegir un numero K de productores
2. Instanciar K productores e IDs y almacenarlos
3. Seleccionar el primer productor y su ID
4. Generar datos sinteticos
5. Enviar los datos sinteticos mediante el productor seleccionado usando como key su ID
6. Seleccionar el siguiente productor en la lista y su ID y repetir los pasos 4 a 5 hasta haber seleccionado todos los productores
7. Repetir infinitamente los pasos 3 a 6
\end{lstlisting}

Los mensajes enviados a un topic con la misma key siempre se asignan a la misma partición. Se asigna el ID del productor a la key a la hora de enviar el mensaje para lograr que los mensajes de un productor siempre vayan a la misma partición.

Esto es debido a que \textit{Kafka} no garantiza un orden global de consumo cuando intervienen múltiples claves y particiones. El orden solo está asegurado dentro de una misma partición y como cada \textit{producer\_id} se envía con una \textit{key} diferente, \textit{Kafka} las distribuye en distintas particiones. Posteriormente el consumidor, lee de varias particiones de forma concurrente, lo que provoca que los mensajes se muestren intercalados. No obstante, dentro de cada partición el orden se mantiene. 

Las modificaciones que se han llevado a cabo en el fichero de producción son las siguientes:

\pagebreak

\begin{vscodePython}
####################
# Codigo original:
####################
# Kafka configuration
kafka_topic = "water_quality"
kafka_bootstrap_servers = ["localhost:9092"] 

# Create Kafka producer
# producer = KafkaProducer(
#     bootstrap_servers=kafka_bootstrap_servers,
#     value_serializer=lambda v: json.dumps(v).encode('utf-8')
# )

###########
# Modificacion para varios productores
###########

producer_number = 7
producers = [] # Aqui almacenaremos nuestros producers
producer_ids = [] # Le damos a cada uno un ID unico
# Create Kafka producer
for i in range(producer_number):
	producer_ids.append("producerID_"+str(i)) # Generacion del ID
	producers.append(KafkaProducer(
			bootstrap_servers=kafka_bootstrap_servers,
			value_serializer=lambda v: json.dumps(v).encode('utf-8')
			)) # Instanciamos el producers


try:
    while True:
        # Iteramos por cada producer (simulacion de multiples fuentes de datos)
        for i, producer  in enumerate(producers):
            # Generate sensor data
            sensor_data = generate_sensor_data()

            # Publish sensor data to Kafka
            # Aniadimos identificador al mensaje
            sensor_data["producer_id"] = producer_ids[i]
            # Aniadimos una key unica (producer_id) para identificar el origen del mensaje
            producer.send(kafka_topic, key=producer_ids[i].encode('utf-8'), value=sensor_data)

            print(f"Sent: {sensor_data}")

        # Wait for 1 second
        time.sleep(1)
except KeyboardInterrupt:
    print("Stopped producing messages.")
finally:
    # Cerramos todos los producers
    [producer.close() for producer in producers]
\end{vscodePython}

\subsection{Consumidores con ID común}
Para poder procesar mensajes con varios consumidores que pertenezcan a un grupo se deben hacer los siguientes cambios:

En primer lugar, debemos tener un almacenamiento que permita agrupar los IDs de sensores de los que vaya a hacerse cargo el consumidor. Para ello vamos a utilizar un diccionario de \textit{python} con una cola que controle el número máximo de elementos que debe almacenar:

\begin{vscodePython}
MAX_SAMPLES = 100
# Almacenamiento dinamico
data_by_sensor = {}
def process_message(message_value):
    """Extrae y almacena los datos del sensor."""
    s_id = message_value['producer_id']
    
    # Inicializacion con deque para manejo automatico de tamanio
    if s_id not in data_by_sensor:
        data_by_sensor[s_id] = {
            "timestamps": deque(maxlen=MAX_SAMPLES),
            **{k: deque(maxlen=MAX_SAMPLES) for k in METRICS.keys()}
        }
    
    storage = data_by_sensor[s_id]
    storage["timestamps"].append(message_value['timestamp'])
    for key in METRICS:
        storage[key].append(message_value[key])
    
    return s_id
\end{vscodePython}

Además de esto, se debe añadir el ID del grupo al consumidor y cómo deserializar el ID de la key:

\section{Integrando procesos ML en data streaming}

\subsection{Carga del modelo e inferencia}
Para la implementación del sistema de toma de \textit{Machine Learning}, se ha desarrollado el siguiente código. La función \textit{perform\_inference} se encarga de transformar el diccionario de datos del sensor en un vector de características compatible con el modelo, extrayendo las variables de temperatura, pH, turbidez y oxígeno disuelto.

\begin{vscodePython}
import joblib
import pandas as pd

MODEL_PATH = "aeration_classifier.joblib"

def load_prediction_model(model_uri):
   """Carga el modelo desde el archivo local mediante deserializacion."""
   try:
      model = joblib.load(model_uri)
      return model
   except Exception as e:
      return None

def perform_inference(model, sensor_data):
   """
   Ejecuta la prediccion y el calculo de densidades de probabilidad.
   Retorna la clase predicha y el vector de probabilidades.
   """
   try:
      input_data = [
      sensor_data["water_temperature"],
      sensor_data["ph_level"],
      sensor_data["turbidity"],
      sensor_data["dissolved_oxygen"]
      ]

       prediction = model.predict([input_data])[0]
       probabilities = model.predict_proba([input_data])[0]
    
       return prediction, probabilities
   except Exception as e:
       return None, None
\end{vscodePython}

Cada vez que llega un nuevo mensaje al tópico se lleva a cabo una inferencia.

\subsection{Envio de predicciones a un nuevo tópico}

El siguiente código imprime por pantalla los detalles de la predicción y envia la información solicitada al siguiente consumidor:

\begin{vscodePython}
for message in consumer:
    sensor_data = message.value
    time_stp = sensor_data["timestamp"]

    # Convertir timestamp a formato legible
    dt = datetime.fromtimestamp(time_stp)
    formatted_time = dt.strftime()

    # Inferencia del modelo
    prediction, probas = perform_inference(model, sensor_data)

    # Etiqueta legible
    label = "NO AERATION" if prediction == 0 else "AERATION"

    # Print limpio
    print(
        f"[{formatted_time}] \t| "
        f"Prediction: {label} ({prediction}) \t| "
        f"Probabilities -> "
        f"NO_AERATION: {probas[0]:.2\%}, "
        f"AERATION: {probas[1]:.2\%}"
    )

    # Envio a Kafka
    send_data = {
        "timestamp": time_stp,
        "prediction": int(prediction),
        "proba_no_areation": float(probas[0]),
        "proba_areation": float(probas[1])
    }

    producer.send(pred_topic, send_data)

\end{vscodePython}
